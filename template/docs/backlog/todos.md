# Backlog de TODOs · Projeto Spark

> Backlog central de pendências que **não bloqueiam** o trabalho atual mas precisam ser registradas pra não se perderem.
> Ordenado por **impacto no andamento**, não por ordem de chegada.
>
> **Como alimentar:** marque inline no doc de origem com `<!-- TODO[data][impacto]: texto -->` e rode `/todo sync`. Ou use `/todo add <texto>` direto aqui.
> **Convivência com `/triage`:** o `/triage` prioriza trabalho de uma rodada (MoSCoW); este backlog é o registro **persistente** de onde os itens vêm e pra onde os WON'T do triage vão descansar.

---

## Legenda de impacto

| Marca | Significado | Quando usar |
|-------|-------------|-------------|
| 🔴 | **Bloqueia fase** | Trava o critério de pronto da fase ativa. Não devia estar aqui por muito tempo — vira trabalho do `/phase-scope`. |
| 🟡 | **Não bloqueia mas importa** | O trabalho anda sem isso, mas há custo real em deixar (dado desatualizado, decisão pendente, débito que cresce). |
| ⚪ | **Nice-to-have** | Melhoria de qualidade/polimento. Só se sobrar tempo. |

**Status:** `aberto` · `em-andamento` · `feito` · `⚠️ órfão` (marcador sumiu do doc mas item seguia aberto — precisa decisão)
